¶1.2 shell

linux不是系统，是个内核，加上CLI等才为系统

因为内核不提供和用户的交互功能，所以需要一个程序提供这种功能，这种程序分为两种：命令行（CLI）和图形界面（GUI），在linux提供CLI的交互程序，我们称之为shell

cat /etc/shells #可以看到多种shell，对于红帽系统来说，默认采用的是bash这个shell

chsh -l #查看系统所有的shell

echo $SHELL #查看当前使用的shell

ps #ps命令用于查看当前进程，可以发现，有一个bash进程

  1. 打开一个终端就可以打开一个shell
  2. 得到一个命令行界面，就触发一个shell

ls /dev/可以看到登录方式

触发一个shell，可以得到一个提示符：

登录用户名@主机名 当前目录

  • #表示的是root用户
  • $表示的是普通用户
  • 方括号里的提示符是由PS1变量展示

echo $PS1 #查看命令提示符的格式

echo $$ #查看当前shell的ID

图形界面中

ctrl+alt+F2 ：切换成了命令行界面 第二个tty

ctrl+alt+F1:默认的图形界面登录

可以通过hostname命令查看主机名

~ 代表当前用户的家目录

¶1.3终端

物理终端：一个键盘、一个鼠标、一个显示器就是一个物理终端

​ 设备文件：/dev/console

虚拟终端（tty）：附加在物理终端之上，以软件方式虚拟出来的终端

​ 设备文件：/dev/tty

模拟终端（pts）: 纯软件实现的终端，比如：在图形界面打开的terminal，使用SSH、telnet等远程打开的界面。

​ 设备文件：/dev/pts/*

系统启动过程中，以物理终端提供

系统启动之后，就以虚拟终端提供

虚拟终端切换

ctrl+alt+f

  • 如果有图形环境，共有六个控制台，图形界面运行在第一个；文本界面运行在第二至第六个。
  • 如果没有线图形界面，则只有5个细腻控制台，全是文本界面。
  • 提示：rhel5之前的版本，都有7个虚拟控制台，前6个为文本界面，第7个为图形界面。

tty #查看当前终端设备

who #查看有哪些用户登录到当前系统

echo aaa > /dev/tty2 #就会在tty2上显示这串字符

echo hello > /dev/pts/5 #在pts5上显示hello

¶1.4命令行

命令 【选项】【参数】

命令：

​ 所谓命令就是一个可执行文件，文件本身包含了可执行的程序代码

​ 输入命令，并回车，就是将这个程序提交给内核，有内核分配资源将其运行起来。运行中的程序需，我们称之为进程。

​ 最左边，必须是一个命令

选项：

​ 用于启用或关闭命令的某个或某些功能

​ 短选项： -表示，可合并使用

​ 长选项： --表示，一般不可合并使用

参数：

​ 命令作用的对象，向命令提供数据。

注意：

  1. ​ 命令，选项，参数中间使用空格隔开
  2. 多上选项之间、多参数之间，使用空格隔开
  3. 一行执行多个命令，使用分号（；）隔开

man+命令查看帮助

在shell中有两类命令：

  1. 内建命令：shell自带，找不到，比如：cd命令，使用which cd 是找不到的
  2. 外部命令：非shell自大，外部的一个可执行文件

type cd #查看一个命令的类型，是内建还是外部

which ls #查看ls命令可执行文件在哪里

whereis ls #查找关键字文件在哪里

使用/usr/bin/ls是没有颜色的，默认ls是‘ls --color=auto’

那为什么无论在什么位置都可以执行命令呢？

echo $PATH

​ #查看PATH变量的值

​ #执行命令时，就会在这个PATH变量指定的路径，从左到右去寻找，一直到找到为止，如果没找到，会提示找不到这个命令

​ #shell搜寻到的外部命令的路径，会被缓存下来，下次就不用再去搜索了

hash

​ #查看缓存的命令

​ #hits. 表示命中次数，也就是被执行了多少次。

hash -r

​ #清除所有缓存

​ #比如：将ls程序文件移到/usr/bin目录下，如果有缓存存在，还会到/bin/下去找ls这个命令，找不到就会报错，这时，就需要清一下缓存，重新去搜寻。

¶1.5 命令演示

shutdown

  1. 作用：关闭计算机，它的使用权限是超级用户。

  2. 格式：

    shutdown [-h] [-i] [-k] [-m] [-t]

  3. 重要参数

    -t：在改变到其它运行级别之前，告诉init程序多久以后关机。

    -k：并不真正关机，只是送警告信号给每位登录者。

    -h: 关机后关闭电源。

    -c：cancel current process 取消目前正在执行的关机程序。所以这个选项当然没有时间参数，但是可是输入一个用来解释的讯息，而这信息将会送到每位使用者。

    -F：在重启计算机时强迫fsck。

    -time：设定关机前的时间。

    -m：将系统改为单用户模式

    -i: 关机时显示系统信息。

  4. 命令说明

    shutdown 命令可以安全地将系统关机。有些用户会使用直接断掉电源的方式来关闭linux系统，这是十分危险的。因为Linux与Windows不同，其后台运行着许多进程，所以强制关机可能会导致进程的数据丢失，使系统处于不稳定的状态，甚至在有的系统中会损坏硬件宿舍（硬盘）。在系统关机前使用shutdown命令，系统管理员会通知所有登录的用户系统将要关闭，并且login指令会被冻结，即新的用户不能再登录。

  5. 举例

    shutdown -h 3:40

    现在关机 shutdown -h now

halt

  1. 作用：关闭系统，它的使用权限是超级用户。

  2. 格式

    halt [-n] [-w] [-d] [-f] [-i] [-p]

  3. 主要参数说明

    -n: 防止sync系统调用，它用在用fsck修补根分区之后，以阻止内核用老版本的超级块覆盖修补过的超级块

    -w：并不是真正的重启或关机，知识写wtmp（/var/log/wtmp）记录。

    -f：没有调用shutdown，而强制关机或重启。

    -i：关机（或重启）前，关掉所有的网络接口。

    -f：强迫关机，不呼叫shutdown这个命令。

    -p：当关机的时候随便做关闭电源的动作。

    -d：关闭系统，但不留下记录。

  4. 命令说明

    halt就是调用shutdown -h。halt执行时，杀死应用进程，执行sync（将存于buffer中的资料强制写入硬盘中）系统调用，文件系统写操作完成后就会停止内核。若系统的运行级别为0或6，则关闭系统；否则以shutdown指令（加上-h参数）来取代。

reboot

  1. ​ 作用：重启计算机，它的使用权限是系统管理者。

  2. 格式

    reboot [-n] [-w] [-d] [-f] [-i]

  3. 主要参数

    -n：在重开机前不做将记忆体资料写回硬盘的动作。

    -w：并不会真的重开机，只要把记录写到/var/log/wtmp/文件里。

    -d：不把记录写到/var/log/wtmp文件里（-n这个参数包含了-d）。

    -i：在重开机之前先把所有与网络相关的装置停止。

exit

  1. ​ 作用：退出系统，它的使用权限是所有用户

  2. 格式

    exit

  3. 参数

    exit命令没有参数，运行后退出系统进入登录界面。

last

  1. 作用：是显示近期用户或终端的登录情况，它的使用权限是所有用户。通过last命令查看该程序的log，管理员可以获知谁曾经或企图连接系统。

  2. 格式

    last [-n] [-f file] [-t tty] [-h 节点] [-l -Ip] [-1] [-y] [1D]

  3. 主要参数

    -n：指定输出记录的条数。

    -f file：指定用文件file作为查询用的log文件。

    -t tty：只显示指定的虚拟控制台上登录情况。

    -h 节点：只显示指定的节点上登陆的情况、

    -i IP：只显示指定的IP上登录的情况。

    -1 ：用IP来显示远端地址。

    -y：显示记录的年、月、日。

    -ID:知道查询的用户名。

    -x：显示系统关闭、用户登录和退出的历史。

id #查看当前用户

root不是真正的超级用户，超级用户为uid为0的用户

uid可以理解为身份证号

hostname #查看当前主机名

pwd #查看当前路径

date #查看日期时间

cal 2016 #查看2016年的日历

¶1.6 TAB补全

​ 输入命令、选项、参数时，可以通过tab键补全功能来减轻输入压力。键入的内容如果足够使其唯一，按一次tab键即可补全，如果键入的内容不够十七唯一，按两次tab键，既可以显示以键入的字符为开头的所有命令。

​ 在RHEL7/CentOS7以前的版本中，bash自动补全功能仅限于命令和文件名，然而网络设备往往可以自动补全命令的参数，常常让搞网络出身的工程师唏嘘不已。

​ 在新出的RHEL7/CentOS7 中的Bash已经可以自动补全命令参数了，需要注意的是RHEL7/CentOS7的默认安装类型是最小安装，他是没有这个功能的。要启用这个功能，你需要安装一个bash-completion包，然后退出bash，重新登陆即可（为重启bash的情况下继续使用可能出现补全功能异常，出现一堆代码）。

RHEL7收费，CentOS免费开源

因为RHEL其中的图标注册了商标，所以收费

bash-2.05及以后的版本提供了自动补齐的编程接口，通过安装bash-completion来使用TAB键自动补齐几乎任何内容，包括参数、文件、目录甚至包名称等等。

rpm -qa | grep bash-completion

¶1.7 命令历史记录

在每个用户的家目录，都有自己的历史记录文件：.bash_history

登陆shell时，会从登陆用户的家目录中，读取.bash_history文件。

登录后，执行的命令历史记录，会先记录到缓存中。

登出时，会将缓存中的历史记录，以追加方式写到.bash_history文件中。

vim /root/.bash_history

history #查看历史命令记录

history -a #立即将缓存中的记录，协会文件

history -c #清除历史

echo $HISTSIZE #查看命令历史记录的条数

echo $HISTFILE #历史文件路径

echo $HISTFILESIZE #命令历史文件记录历史的条数

echo $HISTCONTROL

​ #历史记录的控制

​ #ignoredups，忽略连续且相同的重复的命令

​ #ignorespace，忽略所有以空白开头的命令

​ #ignoreboth，上面两项全部

!数字 #调用历史记录中对应编号的命令

!字符串 #调用历史记录中最近的一个以指定字符串开头的命令

!! #调用上一条命令

!$ #表示上一个命令的最后一段字符

上、下箭头 #可以上下查阅历史记录

左、右箭头 #在当前命令行中左右移动光标

esc+. #将上一命令的最后一个参数复制到当前命令行光标处,重复使用，将继续到更早的命令。

alt+. #将上一个命令的最后一段字符复制到当前，重复使用。

ctrl+r #在历史记录列表中搜索某一模式的命令
