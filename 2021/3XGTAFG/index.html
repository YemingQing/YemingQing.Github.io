¶5.1用户

​ linux系统是一个多用户的系统，同一个账号可以多次同时登陆，对每一个进程而言，也有一个拥有者

3中类型的用户:

  1. 超级管理员，root
  2. 系统用户，服务和内核态进程对应的用户，可以通过ps -ef 查看，比如：ps -ef | grep httpd，可以看到httpd有一个系统用户，叫apache
  3. 普通用户

UID

系统通过UID去区分用户，不是用户名，每一个用户都有一个唯一的标识：

0，超级管理员，可以用id root查看

1-999，系统用户，在rhel7.0之前是0-499

1000及以上，普通用户，7.0之前是500及以上

定义文件：/etc/login.defs

用户信息保存在哪里？

/etc/passed

保存的用户信息，每个用户一行，以冒号分割成7列：

  1. 用户名
  2. 都是x，这是之前的密码保存位置，不安全，所以后来把密码分开存放
  3. 用户的UID
  4. GID，组ID，这个用户的主要组ID，一个用户只能属于一个主组，但可以属于多个附加组
  5. 这个用户的描述信息，可以用 usermod -c “adfasdf” tt为用户tt添加描述信息
  6. 用户的家目录
  7. 该用户的shell类型

/sbin/nologin类型为不允许登录交互

就要设置： -s

密码存在哪里？

/etc/shadow

以冒号分割成9列：

  1. 用户名

  2. 密码

    #经过MD5生成的一个hash值，hash不能倒推，但加密可以解密。hash值一般用于身份验证。

    #如果此处是两个！，表示这个用户没有设置密码

    #passed -l tt 锁定这个用户密码，会在hash值之前多两个！，这时除了root用户，在其他用户下去用这个用户去登录，就不允许登录，因为PAM模块规定了，当root用户切换到其他用户时，不需要验证密码，而普通用户需要。

    #passed -u tt 解锁

  3. 上次更改密码的日期，这个时间是冲1970年1月1日算起

  4. 最短密码期限，从最近一次更改密码的日期算起，再过多少天之后才能再次更改密码，0表示无最短期限，随时可以改

  5. 最长密码期限，必须在这个期限内跟改密码，否则过了这个期限，密码将过期

  6. 密码警告期限，密码过期前多少天开始提醒用户更改密码，0表示不提醒

  7. 密码过期之后的宽限天数，也就是说，密码过期之后，在这个期限内，密码还是可以用的，但如果过了这个期限，密码也就真的失效了，只能找管理员改密码了

  8. 账号到期时间，也是从1970年1月1日算起

  9. 保留

命令行更改密码过期时间

chage -m 1 -M 10 -W 7 -I 17 tt

cat /etc/shadow

解释：1天之内不能修改你密码，10天之后密码到期，密码到期前7天提醒，宽限17天，如果要改默认值，可以到/etc/login.defs里修改

¶5.2 用户

useradd 创建用户

-c 添加描述信息

-g 指定主要组

-G 指定附属组

-d 指定家目录

-s 指定shell

-u 指定UID，注意，只能使用1000以上，并且不能和其他UID冲突

usermod 更改用户信息

-g 更改主要组

-G 更改附属组，如果之前已经指定过附属组，会把该用户从之前附属组中删除

-a 和-G结合使用，不删除之前的组，如果要加入多各组，用空格隔开

-d 更改用户的家目录，但不会将旧家目录中的文件移过去

-m 和-d结合使用，更改家目录时，文件一期一过去

-s 更改用户的shell

-L 锁定用户账号

-U 解锁用户账号

userdel 删除一个用户

-r 同时删除用户和家目录，如果不加这个选项，之删除用户，而家目录还在

id 显示用户信息

字母小写大写标点符号最好

修改用户密码

passwd

passwd

​ #更改当前用户的密码

passwd user1

​ #更改user1这个用户的密码，但这种方式只有root才有权限

echo “redhat” |passed --stdin sam

创建用户时，会自动创建一个和用户名一样的用户组

groupadd

​ -g 指定GID，注意，1000以上，不能冲突

groupmod

​ -n 更改用户组名称

groupdel

gpasswd

​ -d 从组中删除用户

¶5.4 切换用户

su

​ #不会切换当前工作目录，就不会去读取用户家目录下的环境变量文件

su -

​ #会切换到用户的家目录，并且读取用户家目录下的环境变量文件

¶5.5 sudo

分权，把root的权限分给普通用户

配置文件：/etc/sudoers,修改这个文件，不要直接使用vi和vim去修改，因为这两个工具不会去检查语法，用visudo工具去编辑，这个工具会间接调用vim

为一个普通用户分权

visudo

​ tt All=(ALL) NOPASSWD:/usr/sbin/fdisk

​ #tt,表示能使用sudo命令的用户

​ #ALL(ALL),第一个ALL，表示允许从任何主机登陆都可以使用sudo命令，第二个ALL，表示使用sudo后，可以使用任何身份来执行命令。

​ #NOPASSED:表示执行后面的明星不需要输入用户密码。如果没有这句，首次执行fdisk命令会提示你输入当前用户的密码，如果5分钟还没输入密码就会自动退出。如果通过验证，下次在使用这个命令就不需要输密码了。

​ #/usr/sbin/fdisk,这是fdisk命令的全路径

普通用户怎么去执行？

sudo fdisk

用户的别名

如果多个用户都要做同样的操作怎么办呢？

做别名，相当于一个组，把多个用户加入这个别名，然后对这个别名分权。

注意：别名不能小写，不许大写

visudo

​ User Alias AA = user1,user2

​ AA ALL=(ALL) NOPASSWD:/usr/sbin/fdisk

为一个用户组分权

visudo

​ %group ALL=(ALL) NOPASSED:/usr/sbin/fdisk

​ #只要属于group这个组的用户都可以使用fdisk这个命令

命令的别名

visudo

​ Cmnd_Alias CM = /usr/sbin/fdisk,/usr/sbin/useradd

​ AA ALL=(ALL)NOPASSED:CM
